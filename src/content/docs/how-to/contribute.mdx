---
title: How to contribute
---

## Ownership & Accountability

* Every platform has a named documentation owner.
* Owners are responsible for accuracy and timely updates.
* Clear escalation path for stale or incorrect docs.

## Contribution model

* Docs live in a central Git repository.
* Contributions via pull requests from platform teams.
* Lightweight review process with defined SLAs.

## Templates & Standards

* Mandatory use of shared doc templates.
* Predefined structure to ensure consistency.
* Linting/checks to enforce basic quality standards.

## Review & Approval

* Minimum one reviewer from the platform team/.
* Optional reviewer from the docs owner group.
* Approval from the reviewers' group is required before publishing.

## Versioning & Updates

* Docs versioned alongside platform changes.
* Breaking changes require doc updates before release.
* Changelog entry mandatory for consumer-impacting changes.

## Feedback Loop

* Inline feedback or issue creation is supported.
* Consumer feedback routed back to platform owners.
* Docs were iterated based on real usage patterns.
